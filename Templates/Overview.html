Taiwan Digital Identity Wallet (TWDIW)

Adopted W3C Standards

  • Verifiable Credentials Data Model v1.1
  • Decentralized Identifiers v1.0

Public Money, Public Code

  • Basically, all software will be open source.
  • Third-party software for issuer, holder, verifier is possible (technically).

DID Method

  • did:key
  • NIST P-256 elliptic curve keys

TWDIW App

  • Use secure enclave/element on the mobile phone
  • Private key is non-exportable
  • No wallet agent or holder service

The TWDIW app works like "non-custodial wallet".

Selective Disclosure

  • Currently, IETF SD-JWT format from issuer

Zero-Knowledge Proof (early research)

  • Aim to generate the proof on the phone
  • POC from zkID team of Privacy Stewards of Ethereum, based on TWDIW's sandbox/testing environment: Circom validation of JWT ES256 tokens
  • Aligned with high-level requirements of EU Age Verification

For some technical details, see my note on zkp.

Trust List

  • Served with API, currently maintained by moda
  • Deployed on Arbitrum Sepolia Testnet
  • List of trusted issuers, verifiers, and possibly wallets

Trust Model

  • Trust begins with TWDIW official app and its trust list.

For some technical details, see my note on twdiw.
